FastAPI is a modern, fast (high-performance), web framework for building APIs with Python 3.8+ based on standard Python type hints. It is built on top of Starlette for the web parts and Pydantic for the data parts. FastAPI is designed to be easy to use, easy to learn, and ready for production out of the box.
Supabase is an open source Firebase alternative. It provides a Postgres database, Authentication, instant APIs, Edge Functions, Realtime subscriptions, and Storage. In this application, we use Supabase to store files in the "knowledge_files" table and handle vector queries.
Webhooks in Supabase allow you to send real-time HTTP POST requests from your database to external services when records are inserted, updated, or deleted. In this project, when a new file is inserted into the "knowledge_files" table with a status of "pending", Supabase sends a webhook payload to our API endpoint `/webhook/knowledge-file`.
Retrieval-Augmented Generation (RAG) is a technique that combines retrieval of documents/data with a large language model (LLM) to generate highly accurate and context-aware responses to user queries. RAG works by first chunking text files into smaller snippets, computing embedding vectors for these snippets, and storing them in a database. When a query arrives, the most relevant chunks are retrieved using vector similarity search, and these chunks are added to the LLM's prompt context to generate the final answer.
This Python FastAPI application acts as a custom ingestion and query service. The ingestion pipeline runs asynchronously upon receiving the webhook request, and the query service provides a simple GET `/query` REST endpoint for real-time semantic Q&A.
